私のJava開発学習ロードマップ

skill java 2019/12/20

はじめに

Javaの勉強法（現在進行中）をいつかまとめたいな..と思っていたので、社内のAdventCalendarを契機にしてこの度重い腰をあげてまとめてみることにしました。そんな決意をした数日後にSpringFestに参加したのですが、そこでまさにjust fitなセミナーがあったため、その内容も紹介しつつ書いていきたいと思います。

なんでJava?

基幹系のシステムに強いなどJavaを使うメリットは沢山あると思います。私の中では、記述がめんどくさい（≒丁寧に書く習慣が身に付く）、本や勉強度合いの目安になる試験が多い（≒言語を理解する勉強材料として最適）、大変なもの重めのもの迷ったら難しいほうをやる（≒他の言語に移るときに楽と思えそう）、という理由があって続けております。

なにからどこまで

私はリフティング型でJavaの勉強を積上げてきたので（長期的なゴールを意識していない）先のことはあまり考えていませんでした。そんなとき、前述のSpringFestのセミナーで今後の良い指針や今いる地点を知ることが出来ました。わかりやすいため、まずはそれを紹介いたします。

日本Springユーザ会の土岐孝平さんのスライドから

入社1年目のプログラミング初心者がSpringを学ぶための手引き

この中の、

この図は、自分がいまどの地点にいるのかとても整理されます。
いきなりSpringで開発！ではなく、現実的な地に足のついたルートだとおもいます。

本記事では図のなかでも一番上、最初の一歩のところの私流やりかたを紹介したいと思います。

1段目Javaの基本～ベースを埋め立てる～

私が勉強材料としたものは主に3つです。

①これならわかる！Java入門講座 / 水口克也

②Java検定3級

③新　解きながら学ぶJava

まず①はゼロから入ったときにとても役立ちました。

会話形式で内容が進むため、読みやすく、理解を助けてくれました。

②は、自分の定着度の確認のために受けましたが、試験対策（過去問演習）をすると、知識が体系化される、基本の用語を抑えられるなど、想像以上の良い勉強材料となりました。

③は少し難しかったですが、章の区切り毎にキーワード穴埋めページが合ったり、問題形式なので、頭を使うようになるのと、何が分かってないのか自己分析にもなりました。

この3種が私のJavaの土台です。

Javaの基本～脇を固める～

次に私がはじめたのは、基本情報技術者試験の勉強です。一見Javaには関係内容ですがJava開発にはおおいに関係するのです

ここでいまだにやって良かったなぁと思う本の紹介をば

①情報処理教科書 基本情報技術者試験のアルゴリズム問題がちゃんと解ける本 第2版

②基本情報技術者午後問題 橋本のわかって解く! 表計算教室

③改訂4版　基本情報技術者 らくらく突破　表計算 (情報処理技術者試験)

④情報処理教科書 出るとこだけ！ 基本情報技術者［午後］

午後試験は、表計算を受ける気満々でいましたが、仲間がJavaで受けたら？と言ってくれらので、勇気をふり絞ってJavaで受けて嬉しいことに満点をとれて自信につながりました。本番は受けませんでしたが表計算の勉強は地味にいまでも活きますね。絶対参照とかExcelの関数で日ごろ使いますし、その概念を先に知っていられたのは大きいです。それと、表計算は試験では年々難化していて、当日頭を使う量やすばやく動かさないといけないのがシビアだったため、ひねくりの少なめなストレートな問題のおおい解けそうなJavaを選びましたね。

基本情報の勉強は渋々やっていたところもありましたが、後々に活きるのでやってよかったなと今では思えます。

応用情報ではとばされてるとこもありますし。

Javaの基本～すばやく解く。実用する～

たぶん、ベースを固めるためであれば、上記のように何冊もやらなくても、理解力が高い人は大丈夫だと思います。私の場合は色んな書き味や色んな角度から説明してもらわないと理解が追いつかない、分かったつもりが多いので、ちょいと多めにやったかもです。

さて、次に私がやったのは、より実用的な知識、初心者から初級者へ、そんな思いで、始めたのが次の3つ

①スッキリわかるJava入門 第2版 スッキリわかるシリーズ

②オラクル認定資格教科書 Javaプログラマ Bronze SE 7/8 スピードマスター問題集

③Java２級検定

④2週間でJava SE Bronzeの基礎が学べる本 徹底攻略シリーズ

①は復習もかねて読みましたが、わかりやすいので、理解も深まりました。

②問題集から入る方が好きなので、これは夢中になって解きました。公式アプリも買ってプチプチ解いていました。

③これはまた体系的に学ぶにはよく、３級よりも高得点をとれたのは嬉しかったです。最後の応用問題満点取れて叫びました（笑）

④この本は本当に分かりやすい。オーバーロードとオーバーライドの違いや、メモリをどうつかってるかとか、

2段目　web周りなどはクラスタで

3段目　そしてSpringに出会う

社会人2年目はSpringとお友達になりました
Springの裏側ではサーブレットやJSPも動いているということ。だからサーブレットなどやっとくのもよかったということ。忘れかけてたけど、

4段目　Springの応用

これはCRUD課題を終え現在がんばってるところ。設計は日頃の業務でやらせてもらってます。概要設計や、分かる画面に関しては詳細設計まで。

来年は3年目になります。

もう若手ぶってはいられないかな。さみしい。

後輩に教えられるくらい、自分の理解を深い所に持って行って、そして余裕のある人になりたいですね。

ここまで

ここまで色んな本を紹介してきました。多分これはスマートじゃないやり方です。色々回り道してきました。もっと最短コースもあります。

けれどいいたいのは、何がどこで役に立つかわからないので、夢中になれることをそのときどきで真剣にやる。そんなモットーでいきています。

そして、知識の数より、できることの数です。耳が痛いような歯がゆいような。出来る事を増やしましょう。Excelですばやく情報を整理できるとか、ファイル名や変数名を他者にわかりやすく付けられるようになったとか、処理能力が上がったとか、毎日毎日人は進化してるように、まずはできることを数えてみたり、昨日より今日の方がよくできた、よかったそう思える毎日を。

私は進化しつつ深化もがんばりたい。浅く広く色んなことやってるけど、どの経験も繋がだったらいいな。

あとJavaとか触れてないと忘れていきます。毎日少しでもやる。ギターの練習ににてますね。コードらぶ

最近はとある１冊を１章ずつ紹介しようと試みてます
もっとアウトプットをしよう

言語への学問的興味も沸いているがIT学者じゃなく、IT技術者なのです

過去の自分に顔向けできるようにまたがんばろう。
